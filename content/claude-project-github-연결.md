claude.ai Project에 GitHub 레포를 연결했을 때, 프롬프트에 대한 답변이 어떻게 만들어지는지 정리. 이 위키의 RAG 렌즈([[위키-설계-결정]] 참고)가 실제로 작동하는 원리다.

## 한 줄 요약

GitHub 연결 = Claude에게 참고 문서를 주는 것. 답변 = Claude 자체 지식 + 그 문서의 결합.

문서만 요약해주는 챗봇도 아니고, 문서를 무시하는 일반 챗봇도 아니다.

## GitHub 연결이 하는 일 — 읽을거리 제공

- 레포를 연결하면 그 파일들이 대화의 **프로젝트 지식**(Project knowledge)으로 들어간다.
- GitHub에서 뭔가를 "실행"하는 게 아니라, 파일 내용을 텍스트로 읽어 Claude가 볼 수 있게 깔아두는 것.
- 파일이 많아 컨텍스트를 초과하면 자동으로 RAG(관련 부분만 검색해 가져오기)로 전환된다.

## 답변이 만들어지는 방식 — 지식 + 문서 결합

Claude는 프롬프트를 받으면 두 소스를 합쳐 답한다.

- **자체 지식**: 학습으로 알고 있는 일반 지식. 문서에 없는 개념 설명은 여기서 나온다.
- **프로젝트 문서**: 연결된 레포의 노트들. 질문이 문서와 관련되면 이를 근거·맥락으로 우선 참조한다.

즉 문서에 없는 질문이면 자체 지식으로, 문서와 겹치는 질문이면 문서를 인용·연결하며 답한다.

## 주의: "실행"과 "실시간 동기화"는 아니다

- Claude가 레포에서 코드를 돌리거나 커밋하는 게 아니다. 연결 시점에 동기화된 파일 내용을 읽을 뿐.
- 레포에 새 커밋을 푸시했다면 프로젝트 설정에서 동기화를 갱신해야 최신 내용이 반영된다.
- 대화 안에서 Claude가 파일을 만들거나 코드를 실행하는 건 별도 샌드박스에서 하는 일이며, 그 결과가 자동으로 레포에 push되지 않는다(push는 별도 작업 — 이 위키의 "임시 clone → 작업 → push → 삭제" 흐름이 그 예).

## 토큰 사용은 늘어나나, 줄어드나

비교 대상에 따라 다르다.

| 비교 | 토큰 사용 |
|---|---|
| 프로젝트 연결 vs 빈 대화 | 더 씀 (문서만큼 추가) |
| 프로젝트 연결 vs 매번 수동 첨부 | 비슷, 편의성은 우위 |
| RAG 전환 후 vs 전체 문서 주입 | 크게 절약 |

- 문서가 컨텍스트에 전부 들어가는 규모(현재 이 위키)에서는 매 턴 그 문서가 함께 처리되므로, 아무것도 연결 안 한 대화보다 토큰을 더 쓴다. 노트 개당 수 KB라 부담은 미미하지만 공짜는 아니다.
- 문서가 커져 컨텍스트를 초과하면 자동 RAG로 전환되어 관련 조각만 가져온다. 이때부터 절약 효과가 생기며, "수천 개까지 기성품 RAG로 충분"이라는 [[위키-설계-결정]]의 판단이 토큰 관점에서도 유효한 이유.
- 실전에서 사용량에 가장 크게 영향을 주는 건 프로젝트 문서보다 대화 길이다. 대화가 길수록 이전 턴 전체가 매번 다시 처리되므로, 주제가 바뀌면 새 대화를 여는 게 더 큰 절약이다.

## 이 위키와의 관계

이 구조가 [[llm-wiki-구조]]의 세 렌즈 중 RAG 렌즈다. 원본(GitHub `content/*.md`)은 그대로 두고, Project 연결이라는 렌즈를 끼우면 같은 파일이 Claude의 참고 지식이 된다. 검색 정밀도가 아쉬워지기 전까지는 이 기성품 RAG로 충분하다는 판단은 [[위키-설계-결정]]에 기록돼 있다.
