# AI 엔지니어링

LLM으로 무언가를 만드는 기술의 네 계층. 아래층을 흡수하며 쌓이는 스택이다.

* [[에이전틱-엔지니어링]] - 네 계층을 묶어 에이전트를 설계·운영하는 상위 실천 (우산 개념)
* [[에이전트-평가-evals]] - pass@1 과대평가를 넘어 일관성·강건성·실패모드를 함께 재는 평가 방법론
* [[에이전트-스킬은-워크플로다]] - Addy Osmani의 Agent Skills 검토: 실행 워크플로, 검증 게이트, 강제력의 한계와 도입법
* [[ai-엔지니어링-4계층]] - 네 계층을 한자리에서 비교하는 상위 지도
* [[프롬프트-엔지니어링]] - 입력 텍스트(지시·예시·형식·역할) 설계로 출력 품질을 높이는 실천 (최하층)
* [[컨텍스트-엔지니어링]] - 추론 시점에 컨텍스트 윈도를 딱 맞는 정보로 채우는 실천 (중간층)
* [[하네스-엔지니어링]] - 모델을 감싸 에이전트로 만드는 실행 발판 (3층)
* [[하네스-자기개선]] - 근미래 RSI는 가중치가 아니라 하네스를 개선하며 진행된다 (Lilian Weng 정리)
* [[하네스의-미래]] - 하네스는 모델 가중치가 아니라 벤더 API·플랫폼으로 흡수된다 — 사라질 것과 남을 것 (전망)
* [[하네스-핸드북]] - 하네스 진화의 병목은 '편집 생성'이 아니라 'behavior localization(어디를 편집할지 찾기)'이다 — 코드와 동기화된 L1–L3 행위 트리로 푸는 접근 (arXiv 2607.13285 논문리뷰)
* [[ai-화면제작-sdlc-토론]] - 당근 Kraft·오늘의집 AI 활용기를 읽고 프론트엔드·바이브코딩·기획/디자이너 세 직군으로 토론해 뽑은 화면 제작 SDLC — 검증 2층·결정 기록·지위 라벨 (플레이북)
* [[skillopt-비판적-리뷰]] - 스킬 문서를 가중치처럼 최적화하는 SkillOpt 논문 비판적 검토 — 헤드라인 +23.5의 실체는 +5.4 (논문리뷰)
* [[aide2-재귀적-자기개선]] - Weco AI가 "최초의 재귀적 자기개선"을 선언했다 — 무너지는 건 본문이 아니라 제목이고, 진짜 기여는 델타 크기가 아니라 평가의 엄격함이다 (비판적 리뷰)
* [[에이전트-자기개선-서베이]] - Schmidhuber 그룹의 self-improvement 서베이를 2축(업데이트 대상 θ/Σ × 변화 신호 𝒟/e/τ)으로 읽는다 — 대부분의 팀이 손댈 수 있는 자리는 스캐폴드뿐이고, 열린 칸조차 평가 없이는 허상이다 (분석)
* [[루프-엔지니어링]] - 완료 조건까지 반복 실행시키는 제어 흐름 설계 (최상층)
* [[그래프-엔지니어링]] - 에이전트 실행을 명시적 그래프(노드·엣지·상태)로 형식화하는 2026 부상 개념 — 루프의 후속(뜻 A)인지, '순환 vs 정적 DAG'는 어느 쪽인지까지 대립을 열어둔다 (개념)
* [[이중-루프-아키텍처]] - "이중 루프"는 실행구조·학습·리플렉션대상 세 축이 한 단어로 뭉쳐 있다 — 축을 분리해 설계 논쟁과 이론 계보를 바로잡는다
* [[claude-code-advisor-도구]] - 결정 지점에만 더 강한 모델을 부르는 실험적 서버 도구 (페어링·비용·캐시 분석)
* [[codex-교차모델-위임]] - Codex를 Claude Code 서브에이전트로 쓰는 교차 모델 위임 — 직접 실행 검증한 설정 절차, 실제 구조, 대가 (플레이북)
* [[orca-ade]] - git worktree로 코딩 에이전트를 병렬 격리하는 ADE — CLI를 직접 뜯어보니 1급 사용자가 에이전트였다, 그 대가는 워크트리마다 다시 짓는 환경 (도구)
* [[marketing-skills-라이브러리]] - 마케팅 스킬 47개·CLI 64개짜리 오픈소스를 "대규모 스킬 라이브러리 설계"로 뜯은 구조 분석 — 공유 컨텍스트 루트·점진 공개·스킬별 eval·귀속 규율 4패턴 vs 리드젠 퍼널·미검증 품질 (도구)
* [[hermes-agent]] - 자가개선형 오픈소스 에이전트와 Claude Code 위임 구조 (Nous Research, 도구)
* [[github-spec-kit]] - 명세 주도 개발(SDD) 툴킷 v0.12.11 해부 — 강제 메커니즘·트레이드오프와 superpowers·ideas-come-true 비교 (도구)
* [[spec-kit-실전-회고-악보복원]] - Spec Kit을 실제 프로젝트에 끝까지 적용한 회고 — 무엇이 됐고, 무엇이 측정조차 안 됐나 (회고)
