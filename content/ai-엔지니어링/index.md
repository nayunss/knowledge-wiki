# AI 엔지니어링

LLM으로 무언가를 만드는 기술의 네 계층. 아래층을 흡수하며 쌓이는 스택이다.

* [[에이전틱-엔지니어링]] - 네 계층을 묶어 에이전트를 설계·운영하는 상위 실천 (우산 개념)
* [[에이전트-평가-evals]] - pass@1 과대평가를 넘어 일관성·강건성·실패모드를 함께 재는 평가 방법론
* [[에이전트-스킬은-워크플로다]] - Addy Osmani의 Agent Skills 검토: 실행 워크플로, 검증 게이트, 강제력의 한계와 도입법
* [[ai-엔지니어링-4계층]] - 네 계층을 한자리에서 비교하는 상위 지도
* [[프롬프트-엔지니어링]] - 입력 텍스트(지시·예시·형식·역할) 설계로 출력 품질을 높이는 실천 (최하층)
* [[컨텍스트-엔지니어링]] - 추론 시점에 컨텍스트 윈도를 딱 맞는 정보로 채우는 실천 (중간층)
* [[하네스-엔지니어링]] - 모델을 감싸 에이전트로 만드는 실행 발판 (3층)
* [[하네스-자기개선]] - 근미래 RSI는 가중치가 아니라 하네스를 개선하며 진행된다 (Lilian Weng 정리)
* [[루프-엔지니어링]] - 완료 조건까지 반복 실행시키는 제어 흐름 설계 (최상층)
* [[claude-code-advisor-도구]] - 결정 지점에만 더 강한 모델을 부르는 실험적 서버 도구 (페어링·비용·캐시 분석)
* [[codex-교차모델-위임]] - Codex를 Claude Code 서브에이전트로 쓰는 교차 모델 위임 — 직접 실행 검증한 설정 절차, 실제 구조, 대가 (플레이북)
* [[hermes-agent]] - 자가개선형 오픈소스 에이전트와 Claude Code 위임 구조 (Nous Research, 도구)
* [[github-spec-kit]] - 명세 주도 개발(SDD) 툴킷 v0.12.11 해부 — 강제 메커니즘·트레이드오프와 superpowers·ideas-come-true 비교 (도구)
