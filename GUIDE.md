# 사용 가이드

셋업([SETUP.md](SETUP.md))을 끝낸 뒤, **매일 어떻게 쓰는지** 정리한 문서입니다.

## 전체 그림

```
노트 작성 (md)  ──git push──►  GitHub 레포 (원본·이력)  ──자동 빌드──►  위키 사이트 (읽기)
     │                              │
  content/*.md                 claude.ai Project 연결  ──►  RAG 질의응답
```

- **원본은 GitHub 레포 하나** (`content/*.md`). 이게 위키·RAG의 공통 소스.
- 로컬에 폴더를 안 둬도 됩니다 (필요할 때만 임시로 clone).

---

## 1. 노트 추가/편집 — 3가지 방법

### A. Claude Code에게 말하기 (가장 편함)
`wiki-note` 스킬을 설치했다면([claude-skill/README.md](claude-skill/README.md)) 어느 폴더에서든:
```
도커 네트워킹 정리해서 위키에 올려줘
회의록 이거 노트로 만들어줘: [내용 붙여넣기]
"llm-wiki-구조" 노트에 RAG 섹션 추가해줘
```
Claude가 임시 clone → 작성 → push → 삭제까지 다 합니다. 로컬엔 아무것도 안 남습니다.

### B. GitHub 웹에서 직접
레포 → `content/` → **Add file → Create new file** → `제목.md` 작성 → **Commit**. 브라우저만으로 끝.

### C. github.dev 편집기
레포 아무 화면에서 **`.` 키** → 브라우저 VS Code로 여러 파일 편집 → 커밋.

> 어느 방법이든 **push되면 1~2분 뒤 사이트에 자동 반영**됩니다.

---

## 2. 노트 작성 규칙

**한 파일 = 한 주제.** 그래야 위키링크와 RAG 검색이 깨끗합니다.

파일 맨 위 front matter:
```markdown
---
title: 노트 제목
tags:
  - 주제분류
---

본문...
```

**위키링크로 연결** — 다른 노트를 `[[파일명]]`으로 참조:
```markdown
자세한 건 [[도커-네트워킹]] 참고. [[쿠버네티스-기초|k8s]] 처럼 표시 텍스트도 지정 가능.
```
아직 없는 노트를 링크해도 됩니다(사이트에서 stub로 표시 → 나중에 채우면 연결됨).

**폴더**로 분류해도 됩니다: `content/dev/도커.md` 처럼. 사이트 좌측 트리에 그대로 반영.

---

## 3. 읽기 & 스냅샷

- **지금 뭐가 있나** → 위키 사이트 방문 (`https://내아이디.github.io/knowledge-wiki/`). 그래프뷰·백링크·검색 제공.
- **특정 시점 상태** → GitHub 레포 **Commits** 탭. 매 커밋이 전체 스냅샷이라 아무 때로나 되돌아가 볼 수 있음.

---

## 4. RAG로 질문하기

claude.ai → **Projects** → 새 프로젝트 → **knowledge**에 이 레포 연결.
그 프로젝트에서 대화하면 내 노트를 근거로 답합니다. 노트를 push로 갱신하면 프로젝트도 최신화됩니다.

> 노트가 수천 개로 커져 검색 정밀도가 아쉬워지면, 같은 `content/` 폴더에 임베딩 스크립트를 얹어 직접 벡터DB로 확장할 수 있습니다(원본은 그대로).

---

## 5. 비공개 노트

레포가 public이라 사이트도 공개입니다. **비밀·개인정보는 넣지 마세요.**
사이트에 안 띄우고 싶은 노트는 `content/private/` 폴더에 두면 빌드에서 제외됩니다.
단 레포 자체엔 남으므로, **진짜 민감한 내용은 아예 이 위키에 넣지 말 것.**

---

## 6. 문제 해결

- **사이트가 안 바뀜** → 레포 **Actions** 탭에서 최신 실행 확인. 초록불이면 배포 완료(캐시로 몇 분 지연될 수 있음).
- **배포 실패 "try again later"** → Pages 전파 지연. 그 실행을 **Re-run** 하면 됩니다.
- **빌드 실패** → 대개 `content/`의 md 문법 오류(front matter `---` 누락 등). 해당 커밋을 고쳐 다시 push.
