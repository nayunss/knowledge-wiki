# wiki-debug — 파이프라인 사후 진단

발행 사이클이 끝난 뒤(또는 실패한 뒤) `_workspace/`의 산출물을 읽고 **무슨 일이 있었나**를 진단한다.
tinker-cookbook의 `/tinker:debug`(학습 실패 진단)에서 차용 — 실행 스킬(wiki-post)과 **분리된** 진단 스킬.

**스크립트 없음.** FAIL 분류(수치/해석/기타)는 정규식이 아니라 판단이 필요하므로 리포트를 읽고 사람이/LLM이 진단한다. fact-checker가 이미 분류를 내놓으면 그걸 신뢰하되, 근거를 리포트에서 직접 확인한다.

## 입력

- `_workspace/` 경로 (기본: 현재 세션 scratchpad의 `wiki-post-workspace/`)
  - `02_factcheck_report.md` — 판정·FAIL 항목·분류·UNVERIFIED
  - `02_editing_report.md` — 윤문 판정
  - `03b_readability_report.md` — 가독성 판정
- 발행 노트의 위키링크 이름 (예: `tinker-cookbook`)

## 절차

### 1. 리포트 3종을 읽고 게이트별 판정을 수집
각 게이트(fact / copy / readability / 병합 게이트)의 PASS·FAIL과 **주요 지적**을 뽑는다. 수정 루프가 몇 번 돌았는지, 어느 FAIL이 어떻게 해소됐는지.

### 2. 팩트 FAIL을 분류
FAIL이 있었으면 셋 중 하나로:
- **수치** — 틀린 값(측정·집계 오류).
- **해석** — 맞는 값 위에 얹은 틀린 인과·비교·일반화(엣지). ⑥이 겨냥하는 실패.
- **기타** — 출처 귀속 범위 오류, 렌더, 형식 등 위 둘이 아닌 것.

FAIL이 없었으면 "FAIL 없음"도 데이터다. **⑦ 상류 전파 여부**(리서치 산출물 `00_research.md`의 오류가 초안까지 갔는지)도 함께 적는다.

### 3. ⚠️ 현재 원장 상태에 재대조 (이 단계가 이 스킬의 존재 이유)
원장 행을 쓰기 **전에** 위키의 현재 `content/위키-하네스.md`를 **다시 읽는다**(로컬 워크스페이스나 기억이 아니라 원격 최신). 확인할 것:
- 실전 기록 표의 **마지막 행**이 무엇인지(내 편이 몇 번째인지).
- ⑥ 상태(측정 중 / 확정) 와 최신 **창 상태** 문단(몇 편 참인지).
- 다른 세션이 그새 편을 추가했는지.

> **실측 경고**: 이 재대조를 건너뛰면 낡은 원장 상태에 행을 써서 push 충돌·중복·순번 오류가 난다(tinker-cookbook 편에서 실제 발생 — turbovec 편이 그새 ⑥을 확정했는데 확정 전 상태로 행을 썼다가 재작성). **원장 행 초안은 항상 최신 원격을 clone/pull한 뒤 만든다.**

### 4. 붙여넣기 직전 상태로 산출
- **실전 기록 표 행** — `| [[노트이름]] | <검증이 잡은 것: 게이트·분류·해소·특이점> |`
- **⑥ 회귀 계측 표 행** — `| [[노트이름]] | 있음(분류)/없음 | <분류 + 근거 한 줄> |`
- **창/회귀 상태 한 줄** — ⑥ 확정 이후면 "회귀 계측(N편 참) — 회귀 없음/있음" 형식. 확정 전이면 창 상태 갱신.
- 표의 **정확한 컬럼 수·구분자**를 현재 원장과 맞춘다(3단계에서 읽은 실제 형식 기준).

## 출력

짧은 진단(게이트별 판정 + FAIL 분류 + ⑦ 전파) + 위 3종 붙여넣기 블록. 원장에 직접 커밋하지 않는다 — 초안만 내고, 실제 반영·발행은 wiki-post Phase 4/5(발행과 같은 커밋)가 한다.

## 호출 방식

- **자동**: wiki-post Phase 5가 원장 기록 직전에 이 스킬로 진단·행 초안을 받는다.
- **수동**: "왜 이 편 FAIL 났어", "원장 행 다시 뽑아줘", "이전 사이클 post-mortem" → `_workspace`가 남아 있으면 이 스킬로 사후 진단.

## 하지 않는 것

- 리포트를 **다시 검증하지 않는다**(그건 wiki-verify). 이미 나온 판정을 읽고 정리·재대조·형식화만.
- FAIL을 관대하게 재판정하지 않는다 — ⑥ 회귀 계측이 오염된다. 리포트의 판정을 존중한다.
- 사실·수치를 새로 만들지 않는다.
