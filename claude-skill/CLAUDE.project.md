# vibe-coding

## 하네스: knowledge-wiki 기술 글 파이프라인

**목표:** IT 전분야 기술 글을 작성(에반젤리스트+CTO 시각)→검증(최신성·팩트·오탈자·윤문)→발행(OKF 위키)까지 자동 처리.

**트리거:** 기술 글 작성+발행이 함께 요청되면 `wiki-post` 스킬을 사용하라 (예: "~에 대해 글 써서 위키에 올려줘"). 완성된 노트를 올리기만 할 땐 `wiki-note` 스킬. "가독성/최종 검수만 다시"는 `wiki-post` Phase 3.5(readability-reviewer)로 부분 재실행. 단순 질문은 직접 응답 가능.

**종속 문서:** 하네스(에이전트·스킬·게이트) 변경 시 위키 `content/위키-하네스.md`와 레포 README의 하네스·비용 섹션을 같은 커밋에서 갱신하라.
